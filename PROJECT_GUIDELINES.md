# PROJECT-GW - Guía de Producto y Estilo

## Contexto del proyecto
- Este proyecto evolucionará a un CRM masivo para equipos de ventas.
- Objetivo principal: que vendedores puedan visualizar y gestionar clientes de forma clara y rápida.
- La pantalla actual (`index.html`) es solo el inicio.

## Referencias visuales
- Carpeta de referencia: `reference/`
- Usar estas imágenes como guía de composición, espaciado y jerarquía visual en futuras vistas.

## Dirección de diseño (obligatoria)
- Estilo general: minimalista.
- Estética principal: liquid glass.
- Navegación: barras/navs con blur al 45%.
- Bordes: esquinas redondeadas.
- Espaciado: paddings delgados, consistentes y respirables.
- Bordes visibles: solo cuando sean necesarios para jerarquía o separación.
- Densidad visual: evitar elementos amontonados; priorizar claridad y aire entre bloques.

## Reglas de implementación UI
- Mantener una jerarquía visual limpia (tipografía, tamaños y contraste).
- Evitar sobrecargar pantallas con demasiados acentos o efectos.
- Priorizar legibilidad y navegación rápida para uso diario del equipo comercial.
- Cualquier nueva pantalla o componente debe respetar esta guía antes de añadir variaciones.

## Nota operativa para futuras iteraciones
- Antes de diseñar o ajustar una vista, revisar este archivo y las imágenes en `reference/`.
- Si se propone una excepción de estilo, debe justificarse por funcionalidad/usabilidad.

## Regla de layout flotante
- No encerrar elementos en cajas o contenedores visuales cuando se pida un diseño suelto/flotante.
